This zip file includes virtually all of the source code in Accelerated C# 2005.  Not every source file contains a static Main() method, therefore, those files will not compile.  For the files that do compile, they can typically be built by invoking the C# compiler (csc.exe) from the command line as shown below:

 

csc /debug+ <filename>

 

Where <filename> is replaced with the C# file you want to compile.  Of course, /debug+ is optional, but I’m assuming you may want to step through the code using cordbg or DbgCLR.

 

In some circumstances, the default library references will not contain everything needed for the compilation.  For example, some of the examples using string globalization require sysglobl.dll.  In those cases, you can simply add the needed references using the /r option to the compiler.

 

You will also notice some files named dirs and sources throughout the tree.  These are used by the Microsoft build tool build.exe. build.exe is used heavily by Microsoft both internally and externally (shipped in the Windows Driver Kit) and is also the tool used to build the Shared Source CLI (sscli) implementation, commonly known as Rotor.  If you’re not familiar with Rotor, it is a free, shared-source implementation of the CLI and much of the code in it is the actual code in the Microsoft CLR implementation.  Coupled with the CLI spec, it is an excellent learning tool to become intimately familiar with the CLR.

 

It is extremely easy to download and build Rotor.  If you have Rotor built on your system, then building all of the book’s sample code is easy.  Simply open a command prompt and after setting up the Rotor environment via env.bat, change to the top level directory of the code tree where the sources.inc file lives and do the following:

 

build -ceg

 

Since Rotor does not contain all of the libraries contained in the .NET Framework distributed by Microsoft, there are a few examples that won’t compile due to missing references.  You can coerce the Rotor environment to use the .NET Framework libraries by simply setting the _NTTREE environment variable to the full path of your framework location.  For example, on my machine that is:

 

_NTTREE=C:\WINDOWS\Microsoft.NET\Framework\v2.0.50727

 

After doing so, all of the examples will build using build.exe.

 

If you don’t want build.exe to build everything, and you would prefer to only build the examples from one chapter, simply go to that directory and execute build.exe from there.

 

Have fun!
